# 登录与注册页分离设计

## 目标

将当前登录/注册混合切换页拆成两个独立页面，让膳哉入口更像正式健康饮食产品，而不是后台登录表单。

## 页面结构

- `/login`：只负责登录，保留演示账号入口，成功后按角色跳转用户端或维护端。
- `/register`：只负责注册，包含昵称、账号、密码、确认密码，注册成功后进入用户首页。
- 两个页面共用认证视觉外壳组件，统一品牌区、健康价值点、页面布局和移动端适配。

## 视觉方向

采用现代轻健康感：明亮底色、真实健康食物图片、深绿品牌色、克制卡片、清晰表单层级。注册页文案强调“建立健康档案”，登录页文案强调“回到今日饮食计划”。

## 交互规则

- 登录页底部提供“创建账号”跳转。
- 注册页底部提供“返回登录”跳转。
- 注册页要求两次密码一致，密码长度不少于 6 位。
- 表单错误显示在表单上方，并通过 Naive UI message 反馈。
- 移动端优先显示表单，再显示品牌说明，避免首屏被图片挤占。
