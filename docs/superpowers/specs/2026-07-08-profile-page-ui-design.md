# 健康档案页界面优化设计

## 目标

健康档案页从普通资料表单升级为“个人健康控制台”。页面要和首页、智能推荐页保持同一套现代轻健康风格，同时让 BMI、每日热量、饮食目标这些指标看起来更专业，体现健康数据对推荐结果的影响。

## 设计方向

采用“轻健康生活方式感 + 少量专业指标”的方向。

- 主色延续浅绿、米白、深绿、谷物黄和番茄红，不引入新的主色体系。
- 页面整体保持明亮、柔和、轻量，不做深色大面板。
- 核心指标使用数字卡片表达，强化可信度。
- 表单仍然高效可编辑，不做复杂多步骤流程。

## 页面结构

### 顶部引导区

顶部改为浅绿色渐变引导区，标题强调“让推荐更懂你的身体”。引导区右侧展示三个小摘要卡：

- BMI：展示当前计算值和状态。
- 每日目标热量：展示档案推算值。
- 推荐偏好：展示当前饮食目标。

这部分对应首页的大视觉语言，但更克制，避免抢走表单编辑的注意力。

### 主体布局

主体采用两栏布局：

- 左侧为档案编辑区，占主要宽度。
- 右侧为健康洞察区，展示实时预览和说明。

移动端改为单栏，先编辑、后洞察。

### 档案编辑区

档案编辑区分为三块：

1. 身体参数：身高、体重、年龄、性别、烹饪时间偏好。
2. 饮食目标：复用现有 `GoalSegment`，保持目标选择一致。
3. 偏好限制：口味偏好、忌口食材、过敏食材。

身体参数区使用更强的卡片化输入布局，每个输入项有清晰标签和单位。性别切换继续使用当前自定义按钮，不恢复 `n-segmented`。

### 健康洞察区

右侧保留 `HealthSummaryCard`，但视觉上要与新页面一致。下方增加“档案如何影响推荐”的轻说明卡：

- 过滤忌口和过敏食材。
- 根据饮食目标调整热量和蛋白质权重。
- 把 BMI、目标热量、烹饪时间用于推荐理由。

说明卡要简短，像产品提示，不像帮助文档。

### 保存反馈

保存按钮放在表单底部，使用统一深绿色主按钮。保存时展示 loading 文案，保存成功沿用当前消息提示。

## 数据和交互

- 不改后端接口。
- 继续使用 `auth.loadProfile()` 加载档案。
- 继续使用 `auth.saveProfile()` 保存档案。
- 表单输入实时参与右侧健康摘要预览。
- 加载失败时继续显示 `n-alert`。
- 保存失败时保留错误提示。

## 验证

实现后需要验证：

- `npm run build` 通过。
- 浏览器打开 `/user/profile` 页面不空白。
- 桌面端无横向溢出。
- 身高、体重、年龄、性别、目标、偏好标签可编辑。
- 保存按钮可触发接口并显示成功或失败反馈。

## 非目标

- 本次不新增健康历史趋势。
- 本次不修改 BMI 计算逻辑。
- 本次不新增后端字段。
- 本次不做多步骤档案向导。
